# قرارات الأمان (Security Decisions)

> **آخر تحديث:** 2025-12-20

---

## ADR-001: اختيار أدوات الفحص الأمني

**السياق:** نحتاج فحوص أمنية آلية على GitHub.

**القرار:** استخدام:
- GitHub Dependency Review (مجاني)
- Secret Scanning على PRs (مجاني)
- CodeQL إذا متاح (يحتاج Advanced Security)

**السبب:**
- متكاملة مع GitHub
- لا تحتاج إعداد خارجي
- تعمل على PRs بدون حظر Push

**البدائل المرفوضة:**
- Snyk: يحتاج حساب خارجي
- SonarQube: معقد للإعداد

---

## ADR-002: نموذج المصادقة

**السياق:** Backend يحتاج نموذج مصادقة آمن.

**القرار:** JWT + OTP للتحقق الثنائي.

**السبب:**
- JWT stateless وقابل للتوسع
- OTP يضيف طبقة أمان

---

## ADR-003: إدارة الأسرار

**السياق:** منع تسريب الأسرار.

**القرار:**
- الأسرار في GitHub Secrets فقط
- .env.example بقيم وهمية
- Secret scan على كل PR

**السبب:** منع التسريب + سهولة الإدارة
